# 🎭 多智能体工作流（v2.0.0 orch-* 时代）

欢迎来到进阶教程！在 ECC v2.0.0 中，多智能体编排有了**质的飞跃** — 引入了 `orch-*` 编排器家族，把"7 步手动串联"压缩为"1 个意图描述"。本课程从基础编排模式到 v2.0.0 的 orchestrator 实战。

:::info 前置知识
建议先完成 [Hello ECC](./hello-ecc) 和 [TDD 大师课](./tdd-masterclass) 两个入门教程。
:::

## 🚀 v2.0.0 新范式：orch-* 编排器

**旧范式（手动串联 4-7 步）：**

```bash
/plan 实现购物车
/tdd --feature=cart
/code-review
/e2e 测试购物流程
```

**v2.0.0 新范式（一句话完成）：**

```bash
/orch-build-mvp "购物车功能，支持优惠券和库存检查"
```

`orch-*` 编排器内部会自动调度：planner → architect → tdd-guide → code-reviewer → e2e-runner，并在每个关键决策点暂停等待你确认。

### orch-* 家族速览

| 命令 | 一句话能力 |
|------|----------|
| `/orch-build-mvp` | 0→1 快速构建 MVP |
| `/orch-add-feature` | 添加独立新功能 |
| `/orch-fix-defect` | 一键修复缺陷闭环 |
| `/orch-refine-code` | 重构优化（性能 + 质量） |
| `/orch-change-feature` | 修改现有功能行为 |
| `/orch-pipeline` | 自定义编排流水线 |

## 💻 设计你的多智能体 Pipeline

在下面的代码实验室中，尝试为一个真实功能设计多智能体工作流配置：

<CodePlayground 
  title="多智能体 Pipeline 配置"
  language="yaml"
  initialCode={`# multi-agent-pipeline.yaml
# v2.0.0 推荐用 /orch-pipeline 直接调用，
# 这里展示底层 pipeline 配置供学习

name: "Feature Development Pipeline"
description: "Standard workflow for new feature development"

stages:
  # Stage 1: Planning (Sequential)
  - name: planning
    agent: planner
    input: "feature requirement"
    output: "implementation plan"
    
  # Stage 2: Development (Sequential per module)
  - name: development
    agent: tdd-guide
    mode: sequential
    modules:
      - "data-model"
      - "business-logic"
      - "api-endpoints"
      - "ui-components"
    
  # Stage 3: Review (Parallel)
  - name: review
    mode: parallel
    agents:
      - code-reviewer       # Code quality
      - security-reviewer   # Security audit
    
  # Stage 4: Validation (Conditional)
  - name: validation
    condition: "review.passed == true"
    agent: e2e-runner
    fallback:
      agent: tdd-guide
      action: "fix issues and re-review"

# Configuration
config:
  max_retries: 3
  timeout_per_stage: "30m"
  notify_on_failure: true

# v2.0.0 替代方案：
# /orch-pipeline --config=multi-agent-pipeline.yaml
# 或直接：
# /orch-build-mvp "feature description"`}
/>
