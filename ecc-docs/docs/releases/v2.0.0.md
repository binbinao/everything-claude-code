# 🎉 ECC v2.0.0 — The Agent Harness Operating System

> **重大里程碑**：v2.0.0 将 ECC 从单一的 Claude Code 配置集合，升级为**跨 AI 框架的操作控制系统**。

## 🌟 核心亮点

ECC v2.0.0 是 2.0 线的稳定版本：ECC 作为**智能体工作的跨框架操作系统**。Claude Code 仍是一等公民；Codex、OpenCode、Cursor、Gemini、Zed、CodeBuddy、Trae、Qwen、Kiro 和纯终端工作流共享相同的技能、规则、钩子、MCP 约定、发布门控和操作员工作流。

- **261 个公开技能**（v2.0.0 发布时）→ **271 个技能**（当前 main，含 v2.0.0 后的增量）
- **ECC 2.0 控制平面基础**：框架中立的会话适配器（`ecc.session.v1`）覆盖 Claude Code、Codex、OpenCode 和 dmux
- **MCP 清单（`ecc.mcp.v1`）**：跨框架 MCP 服务器配置的统一视图，支持碎片化和漂移检测及密钥脱敏
- **Worktree-lifecycle 服务**：确定性冲突预测与并行智能体 worktree 的安全 GC
- **`orch-*` 编排器家族**（6 个）+ `epic-*` 大型项目工作流（7 个）
- **Rollout 衍生优化包**：`parallel-execution-optimizer`、`benchmark-optimization-loop`、`data-throughput-accelerator`、`latency-critical-systems`、`recursive-decision-ledger` 等

## 🔧 自 rc.1 以来的硬化修复

自 rc.1 到稳定版大约合并了 30 个 PR 的修复。重点：

- **插件钩子在 Node 21+ 上静默失败** (#2184)。钩子运行器依赖 `node -e` 下的 `require.main`，新版 Node 将其留为 undefined — 每个插件钩子干净退出但没有运行。如果你使用的是 Node 21 或更新版本，请立即更新。
- **Windows 可靠性**：`CLAUDE_PLUGIN_ROOT` 路径规范化 (#2139)，通过 stdin 传递提示符以免 shell 破坏 (#2174)，损坏符号链接和 chmod 测试保护 (#2171, #2176)。
- **安全性**：curl 凭证不被放入 argv (#2175)，gateguard 现在将 force/path checkouts 视为破坏性 (#2158)，并提供日常命令门控的环境旋钮 (#2161)，咨询接收硬化。
- **正确性**：会话结束摘要不再破坏用户消息中的 `$`-序列 (#2180)，项目检测在边界上匹配 package keys，让 `preact` 不再被读作 `react` (#2181)，安装清单打包缺口封闭 (#2172)，损坏的旧命令 shim 安全截断 (#2167)。
- **更精简的默认值**：更小的 OpenCode 安装表面与门控的 hooks-runtime (#2140)，`rules/zh` 移出始终加载的默认安装 (#2170)。
- **新表面**：`kubernetes-patterns` 技能 (#2178)，worktree-lifecycle 服务 (#2164)，MCP 清单 (#2146)，codex-worktree 和 opencode 会话适配器 (#2145)，`orch-*` 家族 (#2153)。

## 🆕 v2.0.0 主要新功能

### 1️⃣ Hermes Operator 系统

**Hermes** 是 v2.0.0 引入的**跨 AI 框架操作员**，提供统一控制层：

- **`ecc.session.v1`** — 框架中立的会话适配器
  - 支持 Claude Code、Codex、OpenCode、dmux
  - 统一会话生命周期管理

- **`ecc.mcp.v1`** — MCP 服务器清单
  - 跨框架统一视图
  - 碎片化和漂移检测
  - 密钥脱敏

- **Worktree-lifecycle service** — 并行 worktree 管理
  - 确定性冲突预测
  - 安全 GC（垃圾回收）

### 2️⃣ orch-* 编排器家族

```
/orch-build-mvp        # 快速构建 MVP
/orch-add-feature      # 添加新功能
/orch-fix-defect       # 修复缺陷
/orch-refine-code      # 重构优化
/orch-change-feature   # 修改现有功能
/orch-pipeline         # 自定义编排管道
```

### 3️⃣ 性能优化包

/plugin marketplace add https://github.com/affaan-m/ECC

### 现有用户升级

```bash
/plugin update ecc@ecc
```

> ⚠️ **重要**：Claude Code 插件系统**不支持**分发 `rules/`，必须手动复制：
>
> ```bash
> git clone https://github.com/affaan-m/ECC.git
> cd ECC
> mkdir -p ~/.claude/rules/ecc
> cp -r rules/common ~/.claude/rules/ecc/        # 必须
> cp -r rules/typescript ~/.claude/rules/ecc/    # 按需
> cp -r rules/python ~/.claude/rules/ecc/        # 按需
> ```

### 2026-06 增量变更（MCP 审计）

v2.0.0 发布后，2026-06 进行了一次 MCP 审计：

- **默认 MCP 连接器从 6 个缩减为 1 个**（`chrome-devtools`）
- 移除的：`github`、`context7`、`exa`、`memory`、`playwright`、`sequential-thinking`
- 替代方案：
  - `github` → `github-ops` skill
  - `context7` → `documentation-lookup` skill
  - `exa` → `exa-search` skill
  - `memory` → harness 原生 extended thinking + web search
  - `playwright` → e2e skills（`e2e-testing` 等）
  - `sequential-thinking` → 已退役
- 需要旧连接器？在 `mcp-configs/mcp-servers.json` 显式 opt-in
