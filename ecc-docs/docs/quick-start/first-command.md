# 🎯 第一个命令

恭喜你完成安装！现在让我们来体验 ECC 最强大的功能——**斜杠命令**。

## 什么是斜杠命令？

斜杠命令是以 `/` 开头的特殊指令，就像在 Slack 或 Discord 中使用的命令一样：

```bash
/plan 实现用户登录功能
```

输入这条命令后，ECC 的 **planner 智能体** 就会被唤醒，帮你制定详细的实施计划！

## 试试 /plan 命令

### 步骤 1：打开你的项目

在 VS Code 中打开任意项目（或创建一个新项目）。

### 步骤 2：输入命令

在 Claude Code 对话框中输入：

```bash
/plan 创建一个待办事项应用
```

### 步骤 3：查看计划

planner 智能体会输出类似这样的计划：

```markdown
# 实施计划：待办事项应用

## 需求重述
- 创建一个待办事项应用
- 支持添加、删除、标记完成

## 实施阶段

### Phase 1: 项目初始化
- 创建项目结构
- 安装依赖

### Phase 2: 核心功能
- 实现添加待办
- 实现删除待办
- 实现标记完成

### Phase 3: UI 完善
- 美化界面
- 添加动画效果

## 风险评估
- LOW: 功能简单，风险较低

**等待确认**：是否按此计划执行？
```

### 步骤 4：确认执行

回复 "是" 或 "开始" 即可让 ECC 开始执行计划！

## 其他常用命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `/plan` | 制定计划 | `/plan 添加搜索功能` |
| `/tdd` | TDD 开发 | `/tdd --feature="login"` |
| `/code-review` | 代码审查 | `/code-review` |
| `/e2e` | 端到端测试 | `/e2e 测试登录流程` |

## 命令组合示例

一个完整的开发流程：

```bash
# 1. 先规划
/plan 实现评论功能

# 2. 确认后，用 TDD 开发
/tdd --feature="comments"

# 3. 代码完成后审查
/code-review

# 4. 端到端测试验证
/e2e 测试评论发布流程
```
